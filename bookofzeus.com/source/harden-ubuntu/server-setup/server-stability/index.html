In rare occasions, it may be beneficial to reboot your server reboot if it runs out of memory. This simple solution can avoid you hours of down time.

To enable the Rebooting on OOM feature, you need to edit the /etc/sysctl.conf file:

sudo vi /etc/sysctl.conf

and add or edit the following lines:

vm.panic_on_oom=1
kernel.panic=10

The vm.panic_on_oom=1 line enables panic on OOM; the kernel.panic=10 line tells the kernel to reboot ten seconds after panicking.
